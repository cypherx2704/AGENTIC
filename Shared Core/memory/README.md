# CypherX Memory Service (WP10)

Principal-scoped agent memory for the CypherX platform: **store**, **retrieve**
(vector search), **by-id** CRUD, **sessions**, and **GDPR bulk wipe**, with usage
metering and Contract-19 quota enforcement. Python 3.12 / FastAPI / uv.

Embeddings are generated by the **llms-gateway** (`POST /v1/embeddings`) with a
deterministic in-process **MOCK fallback** so the service (and its tests) run with no
network. Vector dimension is fixed at **1536**.

## Key invariants

* **Principal-only by default** — a memory is owned by the `(principal_type,
  principal_id)` that stored it. A `principal_only` memory is **never** visible to another
  principal in the tenant, under any policy. This is the cross-end-user leak guard. A
  `tenant_shared` memory crosses to other principals only when the tenant's
  `user_scope_visibility` is `tenant` (default `isolated`).
* **404, not 403** for a memory the caller can't see (anti-existence-leak).
* **Idempotency-Key short-circuits BEFORE embedding** — a replay never re-embeds.
* **Dedup ≥ threshold (default 0.95) → bump-only** — a near-duplicate store bumps the
  existing memory's `last_accessed_at` + `score` instead of inserting a copy.
* **RLS tenant isolation** — every tenant-scoped query runs under `app.tenant_id`; the
  runtime role `mem_user` does not bypass RLS.
* **Fail-open** quota/rate enforcement and revocation mirror — availability wins.

## Endpoints

| Method + path                | Scope       | Purpose                                            |
|------------------------------|-------------|----------------------------------------------------|
| `POST /v1/memories`          | `mem:write` | Store a memory (idempotent; 16 KiB cap; dedup-bump)|
| `POST /v1/memories/search`   | `mem:read`  | Vector search (two-pass; `top_k` ≤ 50; filters)    |
| `GET /v1/memories/{id}`      | `mem:read`  | Fetch by id (404 when invisible)                   |
| `PUT /v1/memories/{id}`      | `mem:write` | Update mutable fields (immutable-field rejection)  |
| `DELETE /v1/memories/{id}`   | `mem:write` | Delete (owner-only; 404 when invisible)            |
| `POST /v1/sessions`          | `mem:write` | Create a session (idempotent; 409 cross-principal) |
| `POST /v1/gdpr/wipe`         | `mem:write` | Bulk-wipe a principal (log + delete + event, 1 txn)|
| `GET /livez` `/readyz` `/metrics` | —      | Health + Prometheus (Contract 7)                   |

Auth is dual-mode (Contracts 1/12/13): EXTERNAL bare agent JWT, or INTERNAL service JWT +
`X-Forwarded-Agent-JWT`. Identity (`tenant_id`/principal) comes only from the JWT. The
WP03 verifier-side revocation mirror runs after signature/iss/aud/exp/scope pass.

## Schema (idempotent migrations, `db/migrations/`)

`tenant_config` (visibility policy + dedup threshold), `memories` (principal columns +
scope/type/tags/content/metadata), `memory_vectors_1536` (`vector(1536)` + HNSW),
`sessions` (keyed by principal), `gdpr_wipe_log`, `outbox`, `pricing` (Contract-19 plan
limits). RLS + grants to `mem_user`. Requires the `vector` (pgvector) + `pgcrypto`
extensions.

## Develop

```bash
python -m uv venv
python -m uv sync
# Run the tests (deterministic, no live infra):
./.venv/Scripts/python.exe -m pytest -q -p no:cacheprovider -o faulthandler_timeout=60
./.venv/Scripts/python.exe -m ruff check .
# Run the service:
python -m memory_service     # honours HOST/PORT (default 0.0.0.0:8000)
```

### Configuration (env, no prefix)

`DATABASE_URL`, `KAFKA_BROKERS`, `VALKEY_URL`, `AUTH_JWKS_URL` / `AUTH_ISSUER_URL` /
`AUTH_PLATFORM_AUDIENCE`, `EMBEDDINGS_BASE_URL` / `EMBEDDINGS_MODEL` /
`EMBEDDINGS_SERVICE_TOKEN`, `EMBEDDINGS_MOCK_FALLBACK` (force the offline embedder),
`CONTENT_MAX_BYTES`, `DEDUP_THRESHOLD`, `SEARCH_TOP_K_MAX`, `TTL_SWEEP_*`, `QUOTA_*`.
See `src/memory_service/core/config.py` for the full set + defaults.

## Docker

docker build -t cypherx/memory-service .
