# CLAUDE.md — bkg (Backend Knowledge Graph)

> A **local-first, incrementally-updated, provenance-tracked** knowledge graph of a **backend codebase** (routes/DTOs/middleware/auth/DB/data-flow), served to AI agents over **MCP with NO LLM on the query path**. Deterministic static analysis is the source of truth; runtime observation and a strictly-capped AI proposer are later layers. First target: **Python FastAPI**. Full design in [docs/updated-plan.md](docs/updated-plan.md); build plan in [docs/bkg-kg-core-build-plan.md](docs/bkg-kg-core-build-plan.md); prior-art research in [docs/bkg-research-incremental-code-kg.md](docs/bkg-research-incremental-code-kg.md).

## What this is (and is NOT)
A **separate product** from `CoreProjects/cypherx-a1/` (the Python engineering-memory service). Different graph (code-structural, not social/temporal), different shape (local CLI+MCP, not a cloud service). Do not mix them.

## The moat (three things, only these)
1. **Backend semantic DEPTH** — auth / DTO fields / middleware chain / env / queue / cron / data flow, not just route→handler.
2. **Universal per-fact provenance + confidence** on every node/edge.
3. **VERIFIED true-incremental correctness** — recompute only value-changed facts via reverse-dep propagation + early cutoff, proven by a **byte-identical rebuild oracle**. (Everyone else does watch→re-parse-changed-file; that is not a moat.)

## Load-bearing invariants (do NOT break)
- **No LLM on the query path.** All reasoning is build-time/deterministic; queries are pure index lookups.
- **The database is hidden behind the `GraphStore` port.** `src/bkg/store/sqlite_store.py` is the ONLY module that imports a DB driver. Everything else uses the `GraphStore` ABC + `open_store()` factory. Swapping stores later = one new class, zero changes elsewhere.
- **`@bkg/protocol` (src/bkg/protocol) is the frozen vocabulary.** Nodes/edges/IR/enums + the canonical serializer. `ai`/`runtime` confidence tiers + `verification_status` are declared but UNUSED in the core.
- **Canonical serialization must be bit-stable** — deterministic key ordering, no wall-clock, repo-relative POSIX paths only, no floats. The whole determinism oracle reduces to `digest(incremental) == digest(rebuild)`.
- **Stable nominal identity** — keys are `route:{file}:{router}:{method}:{path}`, `handler:{file}#{symbol}`, etc. — NEVER byte offset / array index.
- **All resolvers must pick the SAME declaration.** Source can redeclare a name (a copy-paste or a mid-edit file parses fine), so *every* query that resolves a model/router by id must agree on which declaration wins — **the first** (`schemaDecl` uses `next(...)`; `schemaDeps` / `schemaBaseCycles` skip an already-seen id; `mountCycles` / `mountChain` take a target's first mount). A resolver that disagrees analyzes a graph the engine never walks: taking the *last* duplicate made the base-cycle guard miss a real cycle (crash) and made `blast_radius` silently return `[]`. Any new by-name resolver must follow first-wins.
- **The analyzer is TOTAL: no parseable input may crash the graph.** tree-sitter parses *invalid* Python by design (the normal state of a file mid-edit), and the engine's dependency graph is a DAG — so every recursive `cx.read` over a source-derived relation (schema bases, router mounts) must be cycle-guarded, and the cycle must **degrade honestly** (`partial`/`inferred`), never be silently hidden behind a plausible-looking answer.
- The incremental engine (P1) is built **before** the real parser. Retrofitting incrementality onto a finished assembler collapses true-incremental (III) into re-parse-changed-file (II).

- **Cycle-safety: the analyzer is now TOTAL (never crashes on parseable input) — DONE.** A pre-existing crash (identical in the old pipeline) took the whole graph down on **cyclic DTOs**: `schemaRef`'s nested-field read forced the nested model's full assembly, so `User.posts: list[Post]` + `Post.author: User` re-entered the engine (`RuntimeError: cycle detected`), killing `graph:all` → every query. Root cause: **a reference graph may be cyclic, but the engine's dependency graph is necessarily a DAG**, so cyclic reachability cannot be expressed as dep-reachability. Fixed by separating the two:
  - **`schemaDecl:{file}:{model}`** (new) — the raw, *non-recursive* per-model declaration. `schema_ref` uses it to check a nested field's DTO **exists** without forcing its assembly (breaking the cycle), and it doubles as a per-model **firewall** (a sibling-model edit backdates to identical bytes).
  - **`schemaDeps:all`** (new) — the whole schema reference graph (bases + nested fields), built in one query with no recursion, hence well-defined on cyclic graphs. `blast_radius` reverse-walks it with a visited set for transitively-affected schemas, then unions those schemas' engine reverse-dep closures to reach endpoints (which is what picks up *every* model-typed param, not just the primary body).
  - **`schemaBaseCycles:all`** (new) + cyclic-mount pruning in `allMounts` — `class A(B)`+`class B(A)` and `a.include_router(b)`+`b.include_router(a)` are invalid but *parseable* (the normal state of a file mid-edit); they now degrade to `partial` / unmounted instead of crashing. tree-sitter parses invalid code by design, so no parseable input may take the graph down.
  - **`blast_radius` no longer reports phantom endpoints** — `reverse_dependencies` walks the store's stale rdep rows (the engine keeps unreachable memo rows and relies on reachability-from-root for liveness), so deleted routes leaked in on a warm engine; the answer is now intersected with the live root.
  Verified: **400/400 states** of a cyclic-DTO determinism fuzz (digest == rebuild, dep-map equality, idempotence, zero cycle errors) and an 18-case blast-semantics matrix (no under- or over-reporting). Blast semantics preserved exactly, incl. transitive nesting and second-model-param edges.
- **Known PRE-EXISTING bugs (surfaced by the cycle-fix adversarial review; reproduced; NOT regressions — each behaves identically in the pre-migration pipeline). Mount resolution + endpoint identity, unrelated to cycles:**
  - **D1 (HIGH) — a re-exported router silently loses its mount prefix and is served `static-certain`.** `app/routers/__init__.py` does `from app.routers.users import router`; `main.py` does `app.include_router(router, prefix='/v1')`. Mount resolution only checks the candidate **file** exists, never that it *declares* that router, so the mount resolves to `app/routers/__init__.py:router` while the route lives at `app/routers/users.py:router` → no `allMounts` match → base case → `prefix=""`. Served: `GET /u`, `static-certain` — the real path is `/v1/u`. Generally: a route on a non-`app` router with no matching mount is indistinguishable from a top-level route. *Fix:* resolve re-export bindings (the candidate file must declare the router), and make `mount_chain`'s base case `partial` for an unmounted non-app router.
  - **D2 (HIGH) — a router mounted twice yields only ONE endpoint.** `app.include_router(r, prefix='/v1')` + `app.include_router(r, prefix='/v2')` is legal FastAPI (two real paths); `mount_chain` returns on the first matching mount, so `/v2/u` is **absent from the graph** with nothing signalling the loss. *Fix:* fan endpoints out across every mount chain (the endpoint id must then include the resolved path), or at minimum mark `partial` when a router has >1 mount.
  - **D3 (MEDIUM) — duplicate route ids emit a duplicate endpoint.** Two handlers on the same `router+method+path` (single framework) → `graph:all` holds the key twice, `list_endpoints` returns two dicts with the **same id**, `trust_summary` counts 2, and `route_fact` takes the first so the second handler is silently dropped. Everything downstream of `graph:all` assumes id-uniqueness. *Fix:* dedup route ids unconditionally in the parser's `_merge` (currently only when >1 framework applies) and flag the collision.
